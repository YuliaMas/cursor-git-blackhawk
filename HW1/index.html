Git-commands

Commands Description
Base
git config --global user.name Yulia вводимо своє ім'я для логування
git config --global user.email yulia.masechko@gmail.com вводимо свій е-мейл для логування
git init ініціалізуємо репозиторій
git status перевіряємо, чи є зміни в локальній версії репозиторія
git add додаємо файл в репозиторій
git commit закомічуємо файл та зміни в ньому
git log показує історію, які є коміти,показує файли і які там відбулись зміни
git checkout вказуємо ще номер id, який копіюємо з попередньої команди git log, можна повернутись в часі і отримати початкові файли та так само повернутись назад до теперішньої версії файлів, також дозволяє переключатись по комітах та гілках
git branch створення нової гілки репозиторію, за замовчуванням вона називається master
git push загрузити всі локальні зміни у віддалений репозиторій
git pull стягнути всі зміни з віддаленого репозиторію собі на локальний
git merge злиття гілок в одну
git clone зклонувати репозиторій собі ,вставивши із github скопіювану url
.gitignore файл файл , за допомогою якого ігноруються інші файли гітом
Advanced
git revert скасовуємо непотрібні чи помилкові коміти
git reset для відміни змін
git rebase для об'єднання змін зроблені в одній гілці з іншою гілкою зі збереженням історії комітів
git cherry-pick спроба накласти поточний коміт на іншу гілку, перенесення коміта по гілках
git stash ховає зміни в файлах і зберігає їх окремо, щоб потім можна було їх повернути
